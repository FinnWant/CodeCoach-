# CodeCoach+ — Project Plan

## Overview
CodeCoach+ helps code coaches collect, analyze, and present student project examples across platforms (Scratch, PixelPad, Unity). This plan provides a step-by-step implementation path, suggested technologies, JSON formats for the three target platforms, testing checklist, and clarifying questions.

## Prerequisites
- Development OS: Windows (already in use). Aim to support macOS/Linux later.
- Tooling: C++17 or later, Qt 6, CMake, SQLite, Git.
- Libraries: `nlohmann/json` (JSON), `libcurl` or Qt network module (HTTP), `OpenCV` (optional image preprocessing), `tesseract` (optional local OCR), or cloud Vision APIs.
- Optional AI APIs: OpenAI (text + vision), Google Cloud Vision, Azure Cognitive Services.

## High-level phases (summary)
1. Requirements & data model (this doc)
2. JSON schema design for Scratch, PixelPad, Unity
3. Project scaffold (C++ backend + Qt UI + DB)
4. Prototype: image upload → OCR → language detection → JSON extraction
5. Full AI integration and heuristics refinement
6. UX: Qt UI with add/search/view flows
7. Tests, docs, and packaging

## Step-by-step Implementation

1) Requirements & scope (1–2 days)
- Confirm target platforms (Scratch, PixelPad, Unity) and expected artifacts (images, code text, project files).
- Decide whether image analysis runs locally or via cloud APIs.
- Choose storage: local SQLite DB + media folder, or cloud storage.
- Deliverable: requirements spec and acceptance criteria.

2) Design JSON schemas (1–2 days)
- Create clear JSON formats for each platform (examples below).
- Define fields: id, title, author, platform, language, description, tags, extracted_code (structured), visual_representation (image references), metadata (creation date, source image).
- Deliverable: `schemas/` directory with `.json` schema files.

3) Project scaffold (1 day)
- Initialize repo with CMake, main app, and tests directory.
- Create `backend/` (C++ core), `ui/` (Qt project), `schemas/`, and `docs/`.
- Add dependency management (vcpkg or include docs for building dependencies).
- Deliverable: minimal app that opens and prints "CodeCoach+ ready".

4) Core backend: data model + DB (2–3 days)
- Implement models mirroring JSON schemas using `nlohmann::json` for serialization.
- Implement SQLite storage layer with migrations.
- Implement media storage (e.g., `media/` folder) and unique filename scheme.
- Deliverable: store/retrieve program entries with JSON payloads.

5) Prototype image pipeline (3–5 days)
- Image ingestion: accept image files (photo or screenshot).
- Preprocess: grayscale, denoise, deskew (OpenCV), optional adaptive thresholding.
- OCR: local Tesseract or cloud Vision. Extract raw text plus bounding boxes.
- Language detection & heuristics: detect if text is Scratch (block-like), PixelPad, or Unity (C#) using heuristics or model.
- Structure extraction: map extracted text into the JSON schema (e.g., Scratch blocks as ordered arrays; Unity objects and C# snippets as code blocks).
- Deliverable: command-line tool `analyze_image <image>` that outputs JSON.

6) AI augmentation & accuracy improvements (3–7 days)
- Use AI to: clean OCR output, detect programming language, infer program purpose, extract notable features, and generate short descriptions.
- Implement prompt templates and fallback heuristics.
- Add QA loop: sample entries reviewed and manual corrections used to improve prompts and rules.
- Deliverable: reliable `analyze_image` with >80% accuracy on sample set (define dataset).

7) CLI + API endpoints (2–4 days)
- Add CLI for adding/listing/searching entries.
- Optional: small local HTTP API (Cpp-REST-SDK or Qt network) for the UI to call.
- Deliverable: working CLI and API docs.

8) Qt UI (5–10 days)
- Wireframes: Add Entry, Search, Browse, Detail view, Edit.
- Implement UI: drag/drop image, form fields auto-filled from analysis, manual edit, save to DB.
- Add image preview and JSON view for advanced users.
- Deliverable: cross-platform Qt app packaging for Windows (MSVC) and optionally others.

9) Testing, docs, packaging (2–4 days)
- Unit tests for parsers, DB layer, and analysis pipeline.
- Integration tests for full flow with sample images.
- Create `README.md`, developer setup, and user guide.
- Build installers or publish portable app bundles.

## Suggested JSON schemas (concise examples)

### Scratch (idea)
{
  "id": "uuid",
  "title": "Move and Collect",
  "platform": "scratch",
  "author": "Student Name",
  "description": "Brief summary generated by AI or coach",
  "tags": ["game","sprite","loops"],
  "visual": {
    "thumbnail": "media/uuid.png",
    "stage": [
      {"sprite": "Cat", "scripts": [ {"when": "greenFlag", "blocks": ["move 10","turn 15"]} ] }
    ]
  },
  "metadata": {"created_at": "2026-03-13T12:00:00Z", "source_image": "media/uuid_src.jpg"}
}

### PixelPad (example structure)
{
  "id": "uuid",
  "title": "Pixel Runner",
  "platform": "pixelpad",
  "classes": [
    {"name": "Player", "type": "Class", "methods": [ {"name":"loop","code":"..."} ] }
  ],
  "assets": {"sprites": ["media/uuid_sprite.png"]},
  "metadata": {"language":"pixelpad","source_image":"media/uuid.jpg"}
}

### Unity (example structure)
{
  "id": "uuid",
  "title": "Simple Platformer",
  "platform": "unity",
  "objects": [
    {"name":"Player","components":[{"type":"Rigidbody2D"},{"type":"PlayerController","code":"public class PlayerController { ... }"}]}
  ],
  "metadata": {"language":"c#","source_image":"media/uuid.png"}
}

## Implementation details & engineering notes
- Use `nlohmann::json` for easy JSON <-> C++ conversion.
- For OCR: local Tesseract is free but less accurate than cloud Vision; cloud Vision reduces infrastructure burden.
- For AI: start with periodic batch processing, then enable near-real-time calls from UI with rate-limiting.
- Privacy: if using cloud APIs, implement opt-in and remove personally identifiable info. Store only what user permits.
- Performance: cache analysis results; perform expensive AI calls asynchronously.

## Testing checklist
- Unit tests for JSON serialization/deserialization.
- Test image pipeline on a variety of photos (good lighting, skewed, mobile photos).
- Manual review and correction flow in UI.
- Security tests for upload endpoints.

## Deliverables (milestones)
- M1: Repo scaffold + schemas + minimal CLI
- M2: Working `analyze_image` prototype
- M3: Qt UI with add/search/view
- M4: Tests, docs, packaging
